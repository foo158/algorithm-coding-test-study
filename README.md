# algorithm-coding-test-study
코딩테스트 대비를 위한 알고리즘과 문제풀이 스터디

<br>

## 스터디 규칙
### 스터디 진행 규칙
1. 매주 정해진 문제를 풀고 풀이 업로드(커밋) 및 해설(PR)을 작성한다.
2. 다른 사람이 올린 풀이(PR)를 읽고 질문, 칭찬할 점, 개선할 점 등을 리뷰하여 코멘트를 남긴다.
3. 스터디 모임 시간에 문제 풀이와 리뷰에 대해 함께 이야기를 나눈다.


### 커밋, PR 규칙
1. 각자 본인의 이름(또는 아이디)으로 된 브랜치를 생성한 뒤 해당 브랜치에 문제 풀이 코드를 추가한다.
2. `solve` 디렉토리 아래에 본인의 이름(또는 아이디) 디렉토리를 추가하고, 그 아래 `boj`/`programmers` 디렉토리에 본인의 풀이를 업로드한다.
3. 커밋 메시지 규칙
   * '[사이트] 문제번호 / 문제이름 - 풀이유형 (언어)' 형식으로 작성한다.
       - 예) [프로그래머스] 12909 / 올바른 괄호 - 스택 (Java), [BOJ] 12865 / 평범한 배낭 - DP (Python)
4. PR 작성 규칙
   * PR 제목은 '[사이트] 문제번호 / 문제이름 - 풀이유형 (언어)' 형식으로 작성한다.
   * PR 내용은 PR 템플릿을 참고하여 작성한다.
5. 스터디 모임을 마치고 각자의 PR을 main 브랜치에 머지한다.
